# API response i18n

The gateway resolves locale from `X-Locale` or `Accept-Language`, supporting `vi` and `en` with Vietnamese as the fallback.

Every proxied JSON response preserves its existing fields and adds:

- `success`: request outcome when the upstream did not provide it;
- `messageCode`: stable machine-readable code;
- `message`: localized user-facing message;
- `locale`: resolved response locale.

Empty list responses use resource-specific codes such as `EMPTY_FEED`, `EMPTY_NOTIFICATIONS`, `EMPTY_COMMENTS`, `EMPTY_FRIENDS`, and `EMPTY_SEARCH`. Clients should prefer their own ARB translation for `messageCode` when available and may use `message` as the server fallback.
